# 02 - 运行框架与启动流程

本章说明 APPLaunch 从进程入口到首页首帧显示的当前路径。重点源码是 `projects/APPLaunch/main/src/main.cpp`、`projects/APPLaunch/main/ui/ui.cpp`、`projects/APPLaunch/main/ui/launcher_ui_runtime.cpp` 和 `ext_components/cp0_lvgl/src/cp0_lvgl_app_runner.cpp`。

## 1. 运行框架概览

APPLaunch 是单进程 LVGL 应用，但 LVGL 初始化、平台服务初始化、主循环等待和退出清理已经由共享的 `cp0_lvgl_run()` 托管：

```text
main()
  -> 配置 Cp0LvglRunOptions
  -> cp0_lvgl_run(options)
      -> lv_init()
      -> cp0_lvgl_init()
      -> options.setup()
          -> 注册 LV_EVENT_KEYBOARD
          -> launcher_ui::init()
          -> ui_screensaver_init()
      -> 强制刷新首帧
      -> 按 lv_timer_handler() 返回值等待 timer 或 wake signal
      -> options.teardown()
          -> launcher_ui::deinit()
      -> 关闭平台服务并 lv_deinit()
```

当前代码不再在 `main.cpp` 中维护固定 `usleep(5000)` 循环，也没有 `APPLaunch_lock()`。外部应用的阻塞执行和 ESC 退出由进程服务负责。

## 2. 关键源码职责

| 路径 | 作用 |
| --- | --- |
| `projects/APPLaunch/main/src/main.cpp` | 配置 APPLaunch 的 setup/teardown 回调并进入共享 runner |
| `ext_components/cp0_lvgl/include/cp0_lvgl_app_runner.hpp` | 声明 `Cp0LvglRunOptions`、`cp0_lvgl_run()` 和通用单页应用模板 |
| `ext_components/cp0_lvgl/src/cp0_lvgl_app_runner.cpp` | LVGL/平台初始化、首帧刷新、事件驱动主循环、统一清理 |
| `projects/APPLaunch/main/ui/ui.cpp` | `launcher_ui::init()` / `deinit()`，管理 Launcher UI 生命周期 |
| `projects/APPLaunch/main/ui/launcher_ui_runtime.cpp` | 创建 theme、`Launch` 和 `UILaunchPage`，构建并显示首页 |
| `projects/APPLaunch/main/ui/ui_launch_page*.cpp` | 首页、轮播、输入、启动动画和启动音 |

## 3. `main()` 与共享 runner

`main.cpp` 当前只负责 APPLaunch 特有的回调：

```cpp
int main(void)
{
    Cp0LvglRunOptions options;
    options.setup = []() {
        if (LV_EVENT_KEYBOARD == 0)
            LV_EVENT_KEYBOARD = lv_event_register_id();
        launcher_ui::init();
        ui_screensaver_init();
        return true;
    };
    options.teardown = []() { launcher_ui::deinit(); };
    return cp0_lvgl_run(std::move(options));
}
```

`cp0_lvgl_run()` 的顺序是：

1. 申请一次性 runner 生命周期并初始化唤醒 semaphore。
2. 调用 `lv_init()`，安装 LVGL resume callback。
3. 调用 `cp0_lvgl_init()` 初始化 display、input 和按配置启用的平台服务。
4. 确认默认 display 已创建，再执行 APPLaunch 的 `setup`。
5. 对 active screen 执行 `lv_obj_invalidate()` 和 `lv_refr_now()`，保证首帧输出。
6. 循环调用 `lv_timer_handler()`；按返回的下一 timer 等待时间休眠，没有 timer 时等待 wake signal。
7. 退出或异常时执行 teardown，随后按顺序关闭平台服务并调用 `lv_deinit()`。

共享 runner 每个进程只允许成功 claim 一次；setup 返回 `false` 或回调抛出异常时会返回非零状态并走清理路径。

## 4. Launcher UI 生命周期

`launcher_ui::init()` 具有重复调用保护：

```text
launcher_ui::init()
  -> 安装全局按键处理器
  -> 创建 LauncherUiRuntime
  -> LauncherUiRuntime::start()
  -> 初始化低电量警告
```

`launcher_ui::deinit()` 执行相反方向的清理：移除全局按键处理器，关闭全局提示、低电量警告、Loading 和 screensaver，然后销毁 `LauncherUiRuntime`。因此新增全局 timer、callback 或异步任务时，应同时补齐 shutdown/deinit 路径。

## 5. `LauncherUiRuntime` 构建首页

构造函数和 `start()` 已明确分工：

```text
LauncherUiRuntime()
  -> create_display()
  -> make_unique<Launch>()
  -> make_shared<UILaunchPage>(launch_.get())
  -> Launch::set_launch_page()

start()
  -> build_launcher_home()
      -> UILaunchPage::create_screen()
      -> Launch::bind_ui()
      -> UILaunchPage::init_input_group()
  -> show_initial_screen()
```

`create_display()` 取得 `cp0_lvgl_init()` 创建的默认 display，并安装 LVGL 默认 theme。字体由共享的 `cp0_fonts()` / `launcher_fonts()` 服务提供，不是 `LauncherUiRuntime` 私有的字体管理对象。

`show_initial_screen()` 在未启用启动动画或 SDL 平台上直接加载首页。设备端启用启动动画时，它通过 `cp0_signal_filesystem_api({"Exists", ...})` 检查 GIF，再选择启动 GIF 或首页；不要绕过平台文件系统 API 直接假设本地路径。

## 6. 首帧、唤醒与退出

- 首帧刷新位于共享 runner，发生在 APPLaunch setup 完成之后。
- `lv_timer_handler()` 返回下一次 timer 的等待毫秒数；runner 使用 timed semaphore wait，`cp0_lvgl_wake()` 可提前唤醒。
- `LV_NO_TIMER_READY` 时 runner 阻塞等待 semaphore，不进行固定周期空转。
- teardown 在 UI 对象仍可安全清理时执行，之后才关闭 sudo、RPC、camera、IMU、audio、PTY、input、WiFi、LoRa、battery 和 LVGL。

## 7. 修改启动流程时的边界

- APPLaunch 特有初始化放入 `options.setup` 或 `launcher_ui::init()`，通用 LVGL 循环改动应在共享 runner 中评估所有使用者。
- 依赖 display 的 UI 创建必须发生在 `cp0_lvgl_init()` 和默认 display 检查之后。
- 新增长生命周期资源时必须定义 teardown 顺序，避免 callback 在 owner 销毁后继续触发。
- 不要在 `main.cpp` 重新加入固定 sleep 主循环或锁文件轮询；这会绕过 runner 的唤醒和清理契约。
